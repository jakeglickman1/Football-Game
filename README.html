Retrobowl Sandlot · Reflection

Project Goal

This term’s focus is to deliver a concise, sandlot-inspired football slice that highlights decision making, crisp ball handling, and a welcoming presentation. Rather than chase full-season simulation, the project emphasizes one polished possession that demonstrates my approach to interaction design and rapid prototyping.

Key Progress

  • Established a lightweight browser build with consistent typography, color, and layout choices inspired by retro stadiums.
  • Implemented route overlays, player markers, and HUD callouts so viewers immediately understand the on-field scenario.
  • Automated a GitHub Pages deployment path, making it easy to distribute updates without manual exports.
  • Captured the current slice through refreshed documentation, screenshots, and status notes.

What Is Going Well

The cohesive look-and-feel is resonating during check-ins, and the limited scope keeps iteration efficient. Control responsiveness continues to improve, and being able to point collaborators to a persistent Pages link has streamlined feedback.

Challenges & Lessons

  • Scope management remains the primary challenge; every new idea competes with the guiding principle of a focused slice.
  • SVG route visualization demanded more planning than anticipated, reinforcing the need for better pre-production sketches.
  • Setting up the GitHub Pages pipeline required learning new workflow permissions but now supports faster reviews.

Plan for the Remainder of the Term

  1. Polish the passing mechanic with better feedback (power meter, catch confirmation, ball flight).
  2. Prototype at least one defensive behavior change so matchups feel dynamic instead of scripted.
  3. Add a short practice/tutorial sequence that introduces controls before the first possession.
  4. Playtest with classmates, capture reactions, and iterate on friction points they surface.

Game Overview

Retrobowl Sandlot is a browser-based football vignette that mixes arcade pacing with a light season loop. Mouse-aimed throws, minimal UI chrome, and lively receiver routes help the experience feel immediate even without a full playbook. Every possession is meant to stand alone as an expressive moment, yet the presentation still nods to television-style HUDs.

Feature Highlights

  • Mouse-aimed throwing that builds power with a hold-and-release mechanic so passes can arc over defenders.
  • Receiver route overlays drawn in SVG to preview WR, TE, and RB assignments before the snap.
  • Animated trenches to hint at pocket dynamics even in this minimalist slice.
  • Assignment-based defense where defenders trail receivers instead of blindly chasing the ball.
  • Persistent LOS and first-down markers so situational awareness is always visible.
  • Eight-week season loop capped by semifinal and championship showdowns, with rings awarded for titles.

Controls at a Glance

Action Input
Move QB / ball carrier Arrow Keys
Sprint Shift
Throw Click + hold to charge, release toward the cursor
Reset season R

Season & Progression

  • Schedule generation: opponents are chosen from a tuned pool with week-based difficulty scaling.
  • Playoffs: winning records unlock semifinal and championship rounds, rewarding trophy rings.
  • Dynamic branding: HUD labels and away end zones update to reflect each opponent.

Development Workflow

  1. Clone or download the repository.
  2. Open index.html in a modern browser—no build step required.
  3. Optionally run python -m http.server or npx serve for live reload during testing.

Deployment Status

  • The repo is GitHub Pages-friendly; pushing to main is enough to serve the prototype.
  • Documentation and this reflection ship through automated workflows that publish README.html.
  • Pages links make it easy for evaluators and peers to view progress without pulling the repo.
